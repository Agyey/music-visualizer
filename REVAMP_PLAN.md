## Executive Summary

### Vision
Become the go-to music video creation tool for South Asian and English-speaking Gen Z/Millennial content creators — a zero-friction, AI-powered platform that transforms any audio file into a shareable, emotionally resonant visual story in under 2 minutes, purpose-built for Reels, Shorts, and TikTok.

### What Was Cut
- **React Native** — replaced with PWA (9/10 agents agreed Three.js can't run in React Native)
- **80% test coverage target** — replaced with 60% backend, 50% frontend (non-renderer)
- **Full design system** — adopt Tailwind incrementally
- **Custom password auth** — OAuth only (Google + GitHub)

### Timeline
- **Total**: 13 weeks (1-week sprints)
- **Phase 0** (Week 1): Security triage
- **Phase 1** (Weeks 2-4): Infrastructure survival
- **Phase 2** (Weeks 5-7): Ship to real users
- **Phase 3** (Weeks 8-13): Scale what works

### Success Metrics
- Time-to-first-video under 2 minutes for 90% of users
- Zero data loss incidents (R2 + PostgreSQL)
- Video render success rate above 95% (async queue)
- 500 WAU within 90 days of Phase 2 launch
- Hindi/multilingual users = 30%+ of base
- Core API p95 latency under 500ms
- Share link CTR above 20%

## Audit Findings

### Feature Map (Current)
| Feature | Status |
|---|---|
| Audio Upload & Analysis (librosa BPM/beats/frequency) | Working |
| AI Transcription (Whisper multilingual) | Working |
| NLP Sentiment/Emotion (HuggingFace) | Working |
| Audio Processing (Demucs stems, EQ, reverb, noise reduction) | Working |
| 4 Web Visualizers (Geometric, Psychedelic, Particle, 3D) | Working, premium quality |
| Video Rendering (PIL/moviepy) | Working, catastrophically slow |
| Live Audio Mode (Web Audio API) | Working |
| User Auth | FAKE — collects passwords, stores nothing |
| Upload History | localStorage only |
| iOS App (Swift/Metal) | Separate, disconnected |

### Critical Issues Found
1. **ZERO test coverage** — no test files exist
2. **Fake auth** — actively deceptive, collects passwords into localStorage
3. **Ephemeral disk** — files destroyed on every Railway deploy (happening NOW)
4. **Synchronous ML** — Whisper/Demucs block the entire API (30-120 seconds)
5. **PIL video renderer** — pixel-by-pixel Python loop, ~20x slower than FFmpeg
6. **No rate limiting** — all endpoints open to abuse
7. **World-readable media** — UUID guessing exposes all files
8. **moviepy 1.0.3** — deprecated, known CVEs
9. **Wildcard CORS** — allow_methods=*, allow_headers=*
10. **790kB bundle** — Three.js not code-split

### Keep vs. Scrap
| Keep | Scrap |
|---|---|
| Pydantic models (clean V2 schema) | Storage layer (local disk) |
| Audio analysis (librosa, solid) | API layer (sync, leaky, no auth) |
| Transcription (Whisper singleton) | Video renderer (PIL pixel loops) |
| NLP analysis (fallback strategy) | Fake auth (UserAuth.tsx) |
| Audio processing (EQ, stems) | iOS app (separate, no shared logic) |
| All 4 renderers (crown jewels) | Docker configs (broken .bak files) |
| VisualizerEngine (multi-canvas) | Inline styles (no design system) |
| Quality Manager (FPS scaling) | |

## Architecture

### System Architecture (Post-Revamp)
```
                    ┌─────────────────────────────────────┐
                    │         Railway (Performance)         │
                    │                                       │
   Users ──HTTPS──▶ │  ┌───────────┐    ┌──────────────┐  │
                    │  │  FastAPI   │◀──▶│  ARQ Worker   │  │
                    │  │  (API)     │    │  (ML + Render)│  │
                    │  └─────┬─────┘    └──────┬───────┘  │
                    │        │                  │           │
                    │        ▼                  ▼           │
                    │  ┌───────────┐    ┌──────────────┐  │
                    │  │   Redis   │    │   Redis      │  │
                    │  │  (Queue)  │◀──▶│   (Cache)    │  │
                    │  └───────────┘    └──────────────┘  │
                    └─────────────────────────────────────┘
                             │                  │
                    ┌────────▼──────┐  ┌───────▼────────┐
                    │  Neon Postgres │  │  Cloudflare R2  │
                    │  (Users/Jobs)  │  │  (Audio/Video)  │
                    └───────────────┘  └────────────────┘

   Frontend: React 18 + TypeScript + Vite 5 + Three.js (code-split)
   Static SPA served from Railway or CDN edge
   OAuth PKCE flow → JWT access + HttpOnly refresh cookie
```

### Database Schema (Neon PostgreSQL)
```sql
-- Table 1: audio_uploads
CREATE TABLE audio_uploads (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  r2_key TEXT NOT NULL,
  original_filename TEXT,
  file_size_bytes BIGINT,
  duration_seconds FLOAT,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Table 2: analysis_results
CREATE TABLE analysis_results (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  audio_id UUID REFERENCES audio_uploads(id) ON DELETE CASCADE,
  analysis_jsonb JSONB NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Table 3: render_jobs
CREATE TABLE render_jobs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  audio_id UUID REFERENCES audio_uploads(id),
  user_id UUID REFERENCES users(id),
  status TEXT CHECK (status IN ('queued','processing','completed','failed')),
  video_r2_key TEXT,
  resolution TEXT,
  aspect_ratio TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  completed_at TIMESTAMPTZ,
  error TEXT
);

-- Table 4: users (added for OAuth)
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT UNIQUE NOT NULL,
  name TEXT,
  oauth_provider TEXT NOT NULL,
  oauth_id TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(oauth_provider, oauth_id)
);
```

### Threat Model (Top 10)
| ID | Threat | Severity | Status |
|---|---|---|---|
| T1 | Credential harvesting via fake auth | CRITICAL | Fix in Sprint 1 |
| T2 | Unauthenticated DoS via upload | CRITICAL | Fix in Sprint 1 |
| T3 | IDOR on /media (UUID guessing) | HIGH | Fix in Sprint 2 |

| T6 | moviepy CVEs | HIGH | Fix in Sprint 4 |
| T7 | Subprocess injection | HIGH | Fix in Sprint 1 |
| T9 | No rate limiting = brute force | HIGH | Fix in Sprint 1 |
| T10 | JWT secret compromise | HIGH | Fix in Sprint 5 |
| T8 | File handle leak → FD exhaustion | MEDIUM | Fix in Sprint 1 |

### Mandatory Security Tickets (10)
- SEC-001: Remove fake auth, implement OAuth + JWT
- SEC-002: File upload limits + magic byte validation
- SEC-003: Presigned URL delivery (replace /media mount)
- SEC-004: Rate limiting (slowapi + Redis)
- SEC-005: CORS explicit allowlist
- SEC-006: Secure subprocess calls
- SEC-007: Dependency audit CI gate
- SEC-008: Render queue concurrency cap
- SEC-009: File handle leak fixes
- SEC-010: JWT secret rotation + secrets management

### Privacy Requirements
- No plaintext passwords stored anywhere
- Minimal PII collection (email + name + OAuth ID only)
- User data isolation (R2 keys namespaced by user_id)
- Right to deletion (purge all user data in one API call)
- Sentry PII scrubbing (before_send hooks)
- Presigned URL TTL: 15 min (renders), 5 min (uploads)
- GDPR/CCPA readiness: privacy policy, deletion mechanism
- Publish SECURITY.md before public launch

## Key Decisions & Disagreements

### Resolved by Judge

| Decision | For | Against | Verdict |
|---|---|---|---|
| React Native mobile | User wanted it | 9/10 agents: Three.js/Metal incompatible | **CUT. PWA instead.** |
| Redis timing | 3 agents: now (task queue) | Cost Optimizer: defer | **Now, queue only** |
| Video rendering | Server FFmpeg vs client captureStream | Both valid | **Both, phased** |
| Database | Railway Postgres | Neon serverless | **Neon free tier** |
| ML hosting | Railway + quantization | Modal.com serverless | **Railway first, Modal if OOM** |
| Coverage target | 80% (original) | 7 agents: unrealistic for WebGL | **60% backend, 50% FE** |

### Engineering Principles (CTO)
1. Stateless API first — no local disk in production
2. Async ML by default — never block the request thread
3. Strict phase sequencing — gates enforced via milestones
4. Test before ship — 60% backend coverage gate before Phase 2
5. Security by default — rate limiting + validation on every endpoint
6. Solo-developer ergonomics — managed services over custom infra
7. Progressive enhancement — lazy load, graceful fallback
8. Observability from day one — Sentry before user features
9. One source of truth for config — env vars only
10. Incremental delivery — each sprint is deployable
